La Etiqueta BUTTON

Volver al inicio

¿Qué es la etiqueta BUTTON?

La etiqueta <button> crea un botón interactivo que puede enviar formularios, ejecutar scripts o simplemente ser clickeable. Es más flexible y semántico que <input type="button">.

Descripcion

La etiqueta <button> representa un botón clickeable. A diferencia de <input type="submit">, puede contener HTML interno (texto, imágenes, iconos), lo que lo hace mucho más versátil para diseño.

Usos comunes de <button>:

  • Envío de formularios
  • Reinicio de formularios
  • Botones de acción (guardar, cancelar, eliminar)
  • Botones con iconos o imágenes
  • Toggles y controles interactivos
  • Botones de navegación
  • Llamadas a acción (Call-to-Action)

Ventaja sobre <input>: Puede contener HTML completo (texto + iconos, imágenes, etc.), mientras que <input type="submit"> solo puede tener texto plano en su atributo value.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

type

Define el tipo de botón. Valores: submit (envía el formulario, por defecto), reset (reinicia el formulario), button (botón genérico sin acción predeterminada, requiere JavaScript). Es importante especificarlo siempre.

Ejemplo: <button type="submit">Enviar</button>

name

Especifica el nombre del botón. Solo relevante para type="submit", ya que el nombre y valor del botón clickeado se pueden enviar al servidor (útil cuando hay múltiples botones de envío).

Ejemplo: <button type="submit" name="accion" value="guardar">Guardar</button>

value

Define el valor que se envía al servidor cuando se hace clic en el botón (solo con type="submit"). A diferencia del texto visible, este es el valor interno enviado.

Ejemplo: <button type="submit" name="opcion" value="1">Opción A</button>

disabled

Atributo booleano que deshabilita el botón, impidiendo que se pueda clickear. Se muestra visualmente atenuado. Útil para estados de carga o validación.

Ejemplo: <button type="submit" disabled>Procesando...</button>

Otros atributos menos comunes pero útiles:

  • form - Asocia el botón con un formulario específico mediante su id (incluso si el botón está fuera del form)
  • formaction - Sobrescribe el atributo action del formulario para este botón específico
  • formmethod - Sobrescribe el método (GET/POST) del formulario para este botón
  • formnovalidate - Deshabilita la validación del formulario solo para este botón
  • autofocus - Coloca el foco automáticamente en el botón al cargar la página

Ejemplo práctico

<!-- Botón de envío básico -->
<button type="submit">Enviar Formulario</button>

<!-- Botón de reinicio -->
<button type="reset">Limpiar Formulario</button>

<!-- Botón genérico (requiere JavaScript) -->
<button type="button" onclick="alert('Hola')">
  Hacer clic
</button>

<!-- Botón con icono y texto -->
<button type="submit">
  <img src="icono-enviar.svg" alt="">
  Enviar Mensaje
</button>

<!-- Múltiples botones de envío con diferentes acciones -->
<button type="submit" name="accion" value="guardar">
  Guardar Borrador
</button>
<button type="submit" name="accion" value="publicar">
  Publicar Ahora
</button>

<!-- Botón deshabilitado -->
<button type="submit" disabled>
  Cargando...
</button>

<!-- Botón que sobrescribe action del form -->
<button type="submit" formaction="/guardar-borrador">
  Guardar Borrador
</button>

Notas importantes

  • Requiere etiqueta de cierre </button>
  • El type por defecto es "submit", así que especifica type="button" si no quieres enviar el form
  • Puede contener casi cualquier HTML (texto, imágenes, iconos) excepto elementos interactivos
  • Preferir <button> sobre <input type="submit"> por su flexibilidad
  • Siempre especificar el atributo type explícitamente para evitar comportamientos inesperados
  • Dentro de un form, un botón sin type se comporta como type="submit"
  • Mejor accesibilidad que usar <div> o <a> estilizados como botones
