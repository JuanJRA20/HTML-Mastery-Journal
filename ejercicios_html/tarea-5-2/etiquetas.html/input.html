La Etiqueta INPUT

Volver al inicio

¿Qué es la etiqueta INPUT?

La etiqueta <input> crea campos de entrada interactivos donde los usuarios pueden ingresar datos. Es uno de los elementos más versátiles de HTML gracias a sus múltiples tipos.

Descripcion

La etiqueta <input> es un elemento vacío (sin etiqueta de cierre) que representa un campo de entrada de datos. Su comportamiento y apariencia varían drásticamente según el valor del atributo type.

Tipos comunes de <input>:

  • text - Campo de texto simple (por defecto)
  • password - Campo de contraseña (oculta los caracteres)
  • email - Campo de correo electrónico (con validación)
  • number - Campo numérico
  • checkbox - Casilla de verificación
  • radio - Botón de opción (selección única en grupo)
  • file - Selector de archivos
  • submit - Botón de envío de formulario
  • date, time, color, y muchos más

Versatilidad: HTML5 añadió más de 10 nuevos tipos de input, haciendo esta etiqueta extremadamente flexible para diferentes necesidades.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

type

Define el tipo de control de entrada. Es el atributo más importante ya que determina completamente el comportamiento del input. Si se omite, el valor por defecto es "text".

Ejemplo: <input type="email">

name

Especifica el nombre del campo que se enviará al servidor. Es esencial para procesar formularios, ya que identifica el valor en el backend. Sin name, el campo no se envía.

Ejemplo: <input type="text" name="nombre_usuario">

value

Define el valor inicial del campo. Para type="text" es el texto predeterminado, para type="checkbox" es el valor que se envía si está marcado, para type="submit" es el texto del botón.

Ejemplo: <input type="text" name="ciudad" value="Madrid">

placeholder

Muestra un texto de ayuda dentro del campo que desaparece cuando el usuario empieza a escribir. Solo funciona con campos de texto. No sustituye a una etiqueta <label>.

Ejemplo: <input type="email" placeholder="tu@email.com">

required

Atributo booleano que hace obligatorio el campo. El formulario no se puede enviar sin completarlo. Activa la validación HTML5 nativa del navegador.

Ejemplo: <input type="text" name="nombre" required>

disabled

Atributo booleano que deshabilita el campo, impidiendo que el usuario interactúe con él. Los campos deshabilitados no se envían con el formulario y aparecen visualmente atenuados.

Ejemplo: <input type="text" name="campo" disabled>

readonly

Atributo booleano que hace el campo de solo lectura. El usuario puede ver pero no modificar el valor. A diferencia de disabled, los campos readonly SÍ se envían con el formulario.

Ejemplo: <input type="text" name="id" value="12345" readonly>

maxlength

Especifica el número máximo de caracteres permitidos en el campo. Solo funciona con campos de texto. El navegador impide escribir más allá de este límite.

Ejemplo: <input type="text" name="codigo" maxlength="10">

min y max

Definen el valor mínimo y máximo permitido para campos numéricos (number, date, time, range). Útiles para validación y controlar rangos de valores.

Ejemplo: <input type="number" name="edad" min="18" max="99">

pattern

Define una expresión regular que el valor del campo debe cumplir para ser válido. Potente herramienta de validación personalizada con HTML5.

Ejemplo: <input type="text" name="codigo_postal" pattern="[0-9]{5}">

Otros atributos menos comunes pero útiles:

  • autocomplete - Habilita (on) o deshabilita (off) el autocompletado del campo
  • autofocus - Atributo booleano que coloca el foco automáticamente en este campo al cargar la página
  • multiple - Permite seleccionar múltiples valores (para type="file" o type="email")
  • step - Define el incremento para campos numéricos (ej: step="0.01" para decimales)
  • checked - Atributo booleano que marca por defecto un checkbox o radio button
  • accept - Especifica tipos de archivo permitidos (solo para type="file")
  • size - Define el ancho visible del campo en caracteres

Ejemplo práctico

<!-- Campos de texto básicos -->
<input type="text" name="nombre" placeholder="Tu nombre" required>
<input type="email" name="email" placeholder="tu@email.com" required>
<input type="password" name="contraseña" minlength="8" required>

<!-- Campo numérico con rango -->
<input type="number" name="edad" min="18" max="120">

<!-- Checkboxes -->
<input type="checkbox" name="newsletter" value="si" checked>
<input type="checkbox" name="terminos" required>

<!-- Radio buttons (mismo name para grupo) -->
<input type="radio" name="genero" value="masculino">
<input type="radio" name="genero" value="femenino">
<input type="radio" name="genero" value="otro">

<!-- Selector de archivos -->
<input type="file" name="documento" accept=".pdf,.doc,.docx">

<!-- Campo de fecha -->
<input type="date" name="fecha_nacimiento" min="1900-01-01" max="2024-12-31">

<!-- Botón de envío -->
<input type="submit" value="Enviar Formulario">

Notas importantes

  • Es una etiqueta vacía (no tiene </input> de cierre)
  • El atributo type determina completamente el comportamiento del input
  • Siempre asociar con <label> para accesibilidad
  • El atributo name es esencial para enviar datos al servidor
  • Los radio buttons del mismo grupo deben tener el mismo name
  • HTML5 añadió validación nativa (required, pattern, type="email", etc.)
  • Para type="file" con múltiples archivos, usa multiple
