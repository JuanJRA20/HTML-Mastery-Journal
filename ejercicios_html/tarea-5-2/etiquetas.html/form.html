La Etiqueta FORM

Volver al inicio

¿Qué es la etiqueta FORM?

La etiqueta <form> crea un formulario interactivo que permite a los usuarios enviar datos al servidor. Es fundamental para la interacción usuario-servidor en la web.

Descripcion

La etiqueta <form> es un elemento de bloque que agrupa controles interactivos como campos de texto, botones, checkboxes, etc., y define cómo se enviarán esos datos al servidor.

Usos comunes de <form>:

  • Formularios de contacto
  • Formularios de registro e inicio de sesión
  • Formularios de búsqueda
  • Formularios de pago y checkout
  • Encuestas y cuestionarios
  • Formularios de comentarios
  • Subida de archivos

Concepto clave: Un formulario por sí solo no hace nada; necesita controles de entrada (<input>, <textarea>, <select>, etc.) y un mecanismo de procesamiento en el servidor o con JavaScript.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

action

Especifica la URL donde se enviarán los datos del formulario cuando se envíe. Si se omite, los datos se envían a la misma página actual. Es uno de los atributos más importantes.

Ejemplo: <form action="/procesar-formulario.php">

method

Define el método HTTP usado para enviar los datos. Valores: GET (datos visibles en la URL, por defecto), POST (datos en el cuerpo de la petición, más seguro). Usa POST para datos sensibles o grandes cantidades de información.

Ejemplo: <form action="/login" method="POST">

enctype

Especifica cómo se codificarán los datos antes de enviarlos al servidor. Solo relevante con method="POST". Valores: application/x-www-form-urlencoded (por defecto), multipart/form-data (requerido para subir archivos), text/plain (raramente usado).

Ejemplo: <form action="/upload" method="POST" enctype="multipart/form-data">

name

Asigna un nombre al formulario, útil para referenciarlo desde JavaScript o cuando hay múltiples formularios en una página. Permite acceder al formulario mediante document.forms.nombreFormulario.

Ejemplo: <form name="contacto" action="/enviar">

Otros atributos menos comunes pero útiles:

  • target - Define dónde mostrar la respuesta (_self, _blank, _parent, _top). Similar al atributo target de los enlaces
  • autocomplete - Habilita (on) o deshabilita (off) el autocompletado de campos. Los navegadores recordarán valores previos
  • novalidate - Atributo booleano que desactiva la validación HTML5 nativa del formulario. Útil si usas validación personalizada con JavaScript

Ejemplo práctico

<!-- Formulario básico de contacto -->
<form action="/enviar-contacto" method="POST">
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre" required>
  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  
  <label for="mensaje">Mensaje:</label>
  <textarea id="mensaje" name="mensaje"></textarea>
  
  <button type="submit">Enviar</button>
</form>

<!-- Formulario de subida de archivos -->
<form action="/upload" method="POST" enctype="multipart/form-data">
  <label for="archivo">Selecciona un archivo:</label>
  <input type="file" id="archivo" name="archivo">
  <button type="submit">Subir</button>
</form>

<!-- Formulario de búsqueda -->
<form action="/buscar" method="GET">
  <input type="search" name="q" placeholder="Buscar...">
  <button type="submit">Buscar</button>
</form>

Notas importantes

  • Requiere etiqueta de cierre </form>
  • Usa method="POST" para datos sensibles (contraseñas, información personal)
  • Usa method="GET" solo para búsquedas o filtros (datos visibles en URL)
  • Para subir archivos, SIEMPRE usa enctype="multipart/form-data"
  • Todos los controles con name se envían al servidor
  • El botón <button type="submit"> o <input type="submit"> envía el formulario
  • HTML5 incluye validación nativa (required, pattern, type="email", etc.)
