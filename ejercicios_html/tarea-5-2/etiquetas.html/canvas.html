La Etiqueta CANVAS

Volver al inicio

¿Qué es la etiqueta CANVAS?

La etiqueta <canvas> proporciona un lienzo (canvas) en el que se pueden dibujar gráficos dinámicamente mediante JavaScript. Es fundamental para crear gráficos, animaciones, juegos y visualizaciones interactivas.

Descripcion

La etiqueta <canvas> es un elemento de HTML5 que crea un área rectangular en la página donde se pueden renderizar gráficos 2D o 3D (con WebGL) usando JavaScript. Por sí solo, el elemento canvas no muestra nada; requiere scripting para dibujar contenido.

Usos comunes de <canvas>:

  • Gráficos y visualizaciones de datos (charts, diagramas)
  • Juegos HTML5 (2D y 3D con WebGL)
  • Animaciones personalizadas
  • Manipulación y edición de imágenes
  • Efectos visuales y partículas
  • Firmas digitales
  • Dibujo colaborativo en tiempo real
  • Renderizado de mapas interactivos

Concepto clave: Canvas es como un papel en blanco. Necesitas usar la API de Canvas con JavaScript para dibujar formas, texto, imágenes, aplicar transformaciones y crear animaciones. Todo el contenido se dibuja mediante código.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

width

Define el ancho del lienzo en píxeles. Si no se especifica, el valor por defecto es 300 píxeles. Es importante establecerlo mediante el atributo HTML, no solo con CSS, para evitar distorsión.

Ejemplo: <canvas width="800"></canvas>

height

Define la altura del lienzo en píxeles. Si no se especifica, el valor por defecto es 150 píxeles. Al igual que width, debe establecerse como atributo HTML para mantener la calidad del dibujo.

Ejemplo: <canvas height="600"></canvas>

Otros atributos menos comunes pero útiles:

  • id - Identificador único para referenciar el canvas desde JavaScript (esencial en la práctica)
  • class - Clase CSS para aplicar estilos al contenedor del canvas

Ejemplo práctico

<!-- Canvas básico -->
<canvas id="miCanvas" width="400" height="300">
  Tu navegador no soporta el elemento canvas.
</canvas>

<script>
// Obtener el canvas y su contexto 2D
const canvas = document.getElementById('miCanvas');
const ctx = canvas.getContext('2d');

// Dibujar un rectángulo rojo
ctx.fillStyle = 'red';
ctx.fillRect(50, 50, 100, 80);

// Dibujar un círculo azul
ctx.beginPath();
ctx.arc(250, 150, 50, 0, Math.PI * 2);
ctx.fillStyle = 'blue';
ctx.fill();

// Dibujar texto
ctx.font = '20px Arial';
ctx.fillStyle = 'black';
ctx.fillText('¡Hola Canvas!', 50, 250);
</script>

<!-- Canvas para animación -->
<canvas id="animacion" width="600" height="400"></canvas>

<script>
const canvasAnim = document.getElementById('animacion');
const ctxAnim = canvasAnim.getContext('2d');
let x = 0;

function animar() {
  // Limpiar canvas
  ctxAnim.clearRect(0, 0, canvasAnim.width, canvasAnim.height);
  
  // Dibujar círculo en movimiento
  ctxAnim.beginPath();
  ctxAnim.arc(x, 200, 30, 0, Math.PI * 2);
  ctxAnim.fillStyle = 'green';
  ctxAnim.fill();
  
  // Actualizar posición
  x += 2;
  if (x > canvasAnim.width) x = 0;
  
  // Siguiente frame
  requestAnimationFrame(animar);
}

animar();
</script>

<!-- Canvas para firma digital -->
<canvas id="firma" width="500" height="200" 
        style="border: 1px solid black;">
</canvas>

<script>
const canvasFirma = document.getElementById('firma');
const ctxFirma = canvasFirma.getContext('2d');
let dibujando = false;

canvasFirma.addEventListener('mousedown', (e) => {
  dibujando = true;
  ctxFirma.beginPath();
  ctxFirma.moveTo(e.offsetX, e.offsetY);
});

canvasFirma.addEventListener('mousemove', (e) => {
  if (dibujando) {
    ctxFirma.lineTo(e.offsetX, e.offsetY);
    ctxFirma.stroke();
  }
});

canvasFirma.addEventListener('mouseup', () => {
  dibujando = false;
});
</script>

Notas importantes

  • Requiere etiqueta de cierre </canvas>
  • El contenido entre las etiquetas solo se muestra si el navegador no soporta canvas
  • Establecer width/height mediante atributos HTML, no solo CSS (evita distorsión)
  • Se necesita JavaScript para dibujar; canvas solo proporciona el área de dibujo
  • Usa getContext('2d') para gráficos 2D o getContext('webgl') para 3D
  • Los gráficos no son escalables (son píxeles, no vectoriales como SVG)
  • Ideal para contenido dinámico; para gráficos estáticos considera SVG
  • Accesibilidad limitada; añade contenido alternativo entre las etiquetas
