La Etiqueta DIV

Volver al inicio

¿Qué es la etiqueta DIV?

La etiqueta <div> (division) es un contenedor genérico de bloque sin significado semántico. Se utiliza para agrupar elementos y aplicar estilos o scripts.

Descripcion

La etiqueta <div> es uno de los elementos más utilizados en HTML. Es un contenedor de propósito general que no tiene significado semántico por sí mismo. Se usa principalmente como estructura para diseño y aplicación de estilos CSS.

Usos comunes de <div>:

  • Agrupar elementos para aplicar estilos CSS
  • Crear estructuras de layout (columnas, grids, contenedores)
  • Envolver secciones para manipulación con JavaScript
  • Organizar contenido visualmente sin implicación semántica
  • Crear componentes de diseño reutilizables

Importante en HTML5: Antes de usar <div>, considera si existe una etiqueta semántica más apropiada como <header>, <nav>, <section>, <article>, <aside> o <footer>. Usa <div> solo cuando no haya alternativa semántica.

Atributos principales

La etiqueta <div> no tiene atributos específicos propios. Acepta todos los atributos globales de HTML5, siendo los más utilizados:

  • id - Identificador único del elemento
  • class - Clase CSS para aplicar estilos (el más común)
  • style - Estilos CSS inline

De hecho, <div> casi siempre se usa con class o id para poder aplicar estilos o scripts específicos.

Ejemplo práctico

<div class="contenedor">
  <div class="encabezado">
    <h1>Título Principal</h1>
  </div>
  
  <div class="contenido">
    <p>Contenido de la página...</p>
  </div>
  
  <div class="sidebar">
    <p>Información adicional</p>
  </div>
</div>

<!-- Mejor alternativa semántica -->
<div class="contenedor">
  <header>
    <h1>Título Principal</h1>
  </header>
  
  <main>
    <p>Contenido de la página...</p>
  </main>
  
  <aside>
    <p>Información adicional</p>
  </aside>
</div>

Notas importantes

  • Es un elemento de bloque (ocupa todo el ancho disponible)
  • No tiene significado semántico, es neutral
  • Preferir etiquetas semánticas de HTML5 cuando sea posible
  • Muy usado con CSS Grid y Flexbox para layouts
  • Por defecto no aplica ningún estilo visual
  • Puede contener cualquier tipo de contenido (bloque o en línea)
  • El equivalente en línea es <span>
