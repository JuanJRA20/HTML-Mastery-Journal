Las Etiquetas AUDIO y VIDEO

Volver al inicio

¿Qué son las etiquetas AUDIO y VIDEO?

Las etiquetas <audio> y <video> permiten incrustar contenido multimedia (audio y video respectivamente) directamente en una página web sin necesidad de plugins externos.

Descripcion

Las etiquetas <audio> y <video> son elementos de bloque introducidos en HTML5 que revolucionaron la forma de incluir multimedia en la web, eliminando la dependencia de tecnologías como Flash.

Usos comunes de <audio>:

  • Podcasts y música
  • Efectos de sonido
  • Narraciones o audiolibros
  • Mensajes de audio

Usos comunes de <video>:

  • Tutoriales y cursos online
  • Presentaciones de productos
  • Testimonios en video
  • Videos de fondo (background videos)
  • Videoclips y trailers

Ventaja principal: Ambas etiquetas ofrecen controles nativos del navegador, compatibilidad multiplataforma y APIs de JavaScript para control avanzado.

Atributos principales

A continuación se explican en detalle los atributos más importantes (compartidos por ambas etiquetas):

src

Especifica la ruta o URL del archivo multimedia. Puede ser una ruta relativa, absoluta o una URL completa. Alternativamente, se pueden usar elementos <source> hijos para proporcionar múltiples formatos.

Ejemplo: <audio src="cancion.mp3"></audio>

controls

Muestra los controles de reproducción nativos del navegador (play, pausa, volumen, progreso). Es un atributo booleano. Sin este atributo, el usuario no podrá controlar la reproducción a menos que uses JavaScript personalizado.

Ejemplo: <video src="video.mp4" controls></video>

autoplay

Inicia la reproducción automáticamente cuando la página carga. Es un atributo booleano. Nota importante: muchos navegadores bloquean autoplay con audio a menos que el video esté silenciado (muted).

Ejemplo: <audio src="musica.mp3" autoplay></audio>

loop

Repite la reproducción del contenido en bucle infinito. Es un atributo booleano. Útil para música de fondo o videos ambientales.

Ejemplo: <video src="fondo.mp4" loop></video>

muted

Silencia el audio por defecto. Es un atributo booleano. Esencial si quieres usar autoplay en videos, ya que los navegadores modernos permiten autoplay solo si el contenido está silenciado.

Ejemplo: <video src="video.mp4" autoplay muted></video>

preload

Indica al navegador cómo debe precargar el archivo. Valores: none (no precargar), metadata (solo metadatos como duración), auto (precargar todo el archivo, por defecto).

Ejemplo: <audio src="podcast.mp3" preload="metadata"></audio>

Atributos específicos de cada etiqueta:

poster (solo para <video>)

Especifica la URL de una imagen que se muestra como vista previa antes de que el video comience a reproducirse. Mejora la experiencia del usuario mostrando una imagen representativa.

Ejemplo: <video src="video.mp4" poster="miniatura.jpg"></video>

width y height (solo para <video>)

Definen las dimensiones del reproductor de video en píxeles. Ayuda al navegador a reservar espacio y mantener la proporción correcta. Si solo se especifica uno, el otro se ajusta automáticamente manteniendo el aspect ratio.

Ejemplo: <video src="video.mp4" width="640" height="360"></video>

Ejemplo práctico

<!-- Audio básico -->
<audio controls>
  <source src="cancion.mp3" type="audio/mpeg">
  <source src="cancion.ogg" type="audio/ogg">
  Tu navegador no soporta audio HTML5.
</audio>

<!-- Video básico -->
<video controls width="640" height="360">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Tu navegador no soporta video HTML5.
</video>

<!-- Video con autoplay (necesita muted) -->
<video autoplay muted loop poster="vista-previa.jpg" width="100%">
  <source src="fondo.mp4" type="video/mp4">
</video>

<!-- Audio con precarga solo de metadatos -->
<audio controls preload="metadata">
  <source src="podcast.mp3" type="audio/mpeg">
</audio>

Notas importantes

  • Ambas etiquetas requieren etiqueta de cierre (no son vacías)
  • Usar múltiples <source> para compatibilidad con diferentes navegadores
  • Formatos comunes de audio: MP3, OGG, WAV
  • Formatos comunes de video: MP4, WebM, OGG
  • El autoplay con sonido está bloqueado en la mayoría de navegadores modernos
  • Siempre incluye texto alternativo entre las etiquetas para navegadores sin soporte
  • Considera el tamaño de los archivos multimedia para optimizar el rendimiento
