La Etiqueta SCRIPT

Volver al inicio

¿Qué es la etiqueta SCRIPT?

La etiqueta <script> se utiliza para insertar código JavaScript en una página HTML o para enlazar archivos JavaScript externos. Es fundamental para añadir interactividad y funcionalidad dinámica a los sitios web.

Descripcion

La etiqueta <script> permite ejecutar código JavaScript en el navegador. Puede contener código directamente entre sus etiquetas (inline) o referenciar un archivo externo mediante el atributo src. JavaScript añade comportamiento dinámico a las páginas estáticas de HTML.

Usos comunes de <script>:

  • Manipulación del DOM (modificar contenido, estilos, estructura)
  • Validación de formularios en el cliente
  • Creación de efectos visuales y animaciones
  • Peticiones AJAX para cargar datos sin recargar la página
  • Gestión de eventos del usuario (clics, scroll, teclado)
  • Integración de librerías externas (React, jQuery, etc.)
  • Análisis y seguimiento (Google Analytics, etc.)
  • Aplicaciones web interactivas (SPAs)

Ubicación recomendada: Tradicionalmente se colocaba en <head>, pero las mejores prácticas modernas recomiendan ponerlo al final del <body> o usar atributos defer/async para optimizar la carga de la página.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

src

Especifica la URL del archivo JavaScript externo que se debe cargar. Cuando se usa src, el contenido entre las etiquetas <script> es ignorado. Es la forma más común y recomendada de incluir JavaScript.

Ejemplo: <script src="app.js"></script>

type

Indica el tipo MIME del script. En HTML5, el valor por defecto es "text/javascript" y ya no es necesario especificarlo. También se usa type="module" para módulos ES6.

Ejemplo: <script type="module" src="main.js"></script>

defer

Atributo booleano que indica que el script debe ejecutarse después de que el documento HTML haya sido completamente parseado. Respeta el orden de aparición. Solo funciona con scripts externos (src). Ideal para scripts que necesitan el DOM completo.

Ejemplo: <script src="script.js" defer></script>

async

Atributo booleano que indica que el script se descargará en paralelo y se ejecutará tan pronto como esté disponible, sin esperar al resto del documento. No respeta el orden. Solo funciona con scripts externos. Útil para scripts independientes como analytics.

Ejemplo: <script src="analytics.js" async></script>

Otros atributos menos comunes pero útiles:

  • integrity - Hash criptográfico para verificar que el archivo no ha sido alterado (seguridad con CDN)
  • crossorigin - Controla las peticiones CORS para scripts de dominios externos
  • nomodule - Atributo booleano que hace que navegadores modernos ignoren el script (fallback para navegadores antiguos)

Ejemplo práctico

<!-- Script inline (código directo) -->
<script>
  console.log('Hola desde JavaScript');
  alert('Bienvenido a la página');
</script>

<!-- Script externo (forma recomendada) -->
<script src="js/app.js"></script>

<!-- Script externo con defer (ejecución diferida) -->
<script src="js/main.js" defer></script>

<!-- Script externo con async (carga asíncrona) -->
<script src="https://analytics.example.com/tracker.js" async></script>

<!-- Script de módulo ES6 -->
<script type="module" src="js/module.js"></script>

<!-- Script desde CDN con integridad -->
<script 
  src="https://cdn.example.com/library.js"
  integrity="sha384-abc123..."
  crossorigin="anonymous">
</script>

<!-- Ubicación típica en el documento -->
<!DOCTYPE html>
<html>
  <head>
    <title>Mi Página</title>
    <!-- Scripts críticos con defer -->
    <script src="critical.js" defer></script>
  </head>
  <body>
    <h1>Contenido</h1>
    
    <!-- Scripts al final del body (forma tradicional) -->
    <script src="js/app.js"></script>
  </body>
</html>

<!-- Ejemplo de manipulación del DOM -->
<button id="miBoton">Haz clic</button>

<script>
  document.getElementById('miBoton').addEventListener('click', function() {
    alert('¡Botón clickeado!');
  });
</script>

Notas importantes

  • Requiere etiqueta de cierre </script> (nunca usar forma autocerrante)
  • Los scripts bloquean el renderizado del HTML mientras se ejecutan (usar defer/async para evitarlo)
  • defer: espera al DOM, mantiene orden - async: ejecuta cuando esté listo, sin orden
  • Preferir scripts externos sobre código inline para mejor mantenimiento y caché
  • Colocar scripts al final del <body> o usar defer para scripts que necesitan el DOM
  • El atributo type="text/javascript" ya no es necesario en HTML5
  • Los scripts inline no aceptan atributos src, defer o async
  • Para seguridad, evitar inline scripts y usar CSP (Content Security Policy)
