La Etiqueta SELECT

Volver al inicio

¿Qué es la etiqueta SELECT?

La etiqueta <select> crea una lista desplegable (dropdown) que permite a los usuarios elegir una o varias opciones de un conjunto predefinido.

Descripcion

La etiqueta <select> es un control de formulario que presenta opciones al usuario en un menú desplegable. Funciona en conjunto con las etiquetas <option> (que definen cada opción) y opcionalmente <optgroup> (para agrupar opciones).

Usos comunes de <select>:

  • Selección de países o ciudades
  • Categorías de productos
  • Opciones de configuración
  • Rangos de fechas o cantidades
  • Filtros de búsqueda
  • Selección de idioma
  • Menús de navegación en dispositivos móviles

Ventaja principal: Ahorra espacio en la interfaz cuando hay muchas opciones, mostrándolas solo cuando el usuario abre el desplegable.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

name

Especifica el nombre del campo que se enviará al servidor. Es esencial para identificar qué opción seleccionó el usuario cuando se procesa el formulario.

Ejemplo: <select name="pais"></select>

multiple

Atributo booleano que permite al usuario seleccionar múltiples opciones manteniendo presionada la tecla Ctrl (o Cmd en Mac). Cambia la apariencia a una lista en lugar de desplegable.

Ejemplo: <select name="intereses" multiple></select>

size

Define cuántas opciones son visibles simultáneamente. Si no se especifica, se muestra como desplegable (dropdown). Con size mayor a 1, se muestra como lista scrollable.

Ejemplo: <select name="opciones" size="5"></select>

required

Atributo booleano que hace obligatoria la selección de una opción. El formulario no se puede enviar sin seleccionar algo. Útil para campos obligatorios como país o categoría.

Ejemplo: <select name="categoria" required></select>

Otros atributos menos comunes pero útiles:

  • disabled - Atributo booleano que deshabilita todo el select, impidiendo la selección
  • autofocus - Atributo booleano que coloca el foco automáticamente en este campo al cargar la página
  • form - Asocia el select con un formulario específico mediante su id

Atributos de la etiqueta OPTION

Las opciones dentro de <select> se crean con <option> y tienen sus propios atributos importantes:

value

Define el valor que se envía al servidor cuando se selecciona esta opción. Si no se especifica, se envía el texto visible de la opción.

Ejemplo: <option value="es">España</option>

selected

Atributo booleano que marca esta opción como seleccionada por defecto cuando se carga la página.

Ejemplo: <option value="mx" selected>México</option>

disabled

Atributo booleano que deshabilita una opción específica, impidiendo su selección. Se muestra atenuada visualmente.

Ejemplo: <option value="sold-out" disabled>Agotado</option>

Ejemplo práctico

<!-- Select básico -->
<label for="pais">País:</label>
<select id="pais" name="pais" required>
  <option value="">-- Selecciona un país --</option>
  <option value="es">España</option>
  <option value="mx">México</option>
  <option value="ar">Argentina</option>
  <option value="co">Colombia</option>
</select>

<!-- Select con opción preseleccionada -->
<select name="idioma">
  <option value="es" selected>Español</option>
  <option value="en">English</option>
  <option value="fr">Français</option>
</select>

<!-- Select con grupos -->
<select name="ciudad">
  <optgroup label="España">
    <option value="madrid">Madrid</option>
    <option value="barcelona">Barcelona</option>
  </optgroup>
  <optgroup label="México">
    <option value="cdmx">Ciudad de México</option>
    <option value="guadalajara">Guadalajara</option>
  </optgroup>
</select>

<!-- Select múltiple -->
<select name="intereses" multiple size="5">
  <option value="deportes">Deportes</option>
  <option value="musica">Música</option>
  <option value="cine">Cine</option>
  <option value="lectura">Lectura</option>
  <option value="viajes">Viajes</option>
</select>

Notas importantes

  • Requiere etiqueta de cierre </select>
  • Solo puede contener <option> y <optgroup> como hijos directos
  • La primera <option> suele ser un placeholder ("-- Selecciona --") con value vacío
  • Siempre asociar con <label> para accesibilidad
  • Usar <optgroup> para organizar muchas opciones en categorías
  • Con multiple, el usuario puede seleccionar varias opciones con Ctrl/Cmd + clic
  • El valor enviado al servidor proviene del atributo value de <option>
