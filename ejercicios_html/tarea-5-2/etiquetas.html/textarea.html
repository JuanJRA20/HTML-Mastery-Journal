La Etiqueta TEXTAREA

Volver al inicio

¿Qué es la etiqueta TEXTAREA?

La etiqueta <textarea> crea un campo de texto multilínea donde los usuarios pueden escribir contenido extenso como comentarios, mensajes o descripciones.

Descripcion

La etiqueta <textarea> es un control de formulario que permite entrada de texto en múltiples líneas. A diferencia de <input type="text">, que es para texto corto de una sola línea, <textarea> es ideal para contenido más largo.

Usos comunes de <textarea>:

  • Campos de comentarios
  • Mensajes de contacto
  • Descripciones de productos o servicios
  • Reseñas y opiniones
  • Biografías de usuarios
  • Notas o anotaciones
  • Código fuente o texto preformateado

Diferencia clave: El contenido inicial se coloca entre las etiquetas de apertura y cierre, no en un atributo value como en <input>.

Atributos principales

A continuación se explican en detalle los atributos más importantes:

name

Especifica el nombre del campo que se enviará al servidor. Es esencial para identificar el contenido del textarea cuando se procesa el formulario.

Ejemplo: <textarea name="comentario"></textarea>

rows

Define el número de líneas visibles del textarea (altura). Controla cuántas líneas de texto se muestran sin necesidad de scroll. El usuario puede escribir más allá de este límite.

Ejemplo: <textarea rows="5"></textarea>

cols

Define el ancho visible del textarea en número de caracteres. En la práctica moderna, es más común controlar el ancho con CSS que con este atributo.

Ejemplo: <textarea cols="50"></textarea>

placeholder

Muestra un texto de ayuda dentro del campo que desaparece cuando el usuario empieza a escribir. Útil para dar instrucciones o ejemplos al usuario.

Ejemplo: <textarea placeholder="Escribe tu mensaje aquí..."></textarea>

maxlength

Especifica el número máximo de caracteres permitidos. El navegador impide escribir más allá de este límite. Útil para limitar la longitud de comentarios o mensajes.

Ejemplo: <textarea maxlength="500"></textarea>

required

Atributo booleano que hace obligatorio el campo. El formulario no se puede enviar sin completarlo. Activa la validación HTML5 nativa.

Ejemplo: <textarea name="mensaje" required></textarea>

Otros atributos menos comunes pero útiles:

  • disabled - Atributo booleano que deshabilita el campo, impidiendo interacción del usuario
  • readonly - Atributo booleano que hace el campo de solo lectura (se puede ver pero no modificar)
  • autocomplete - Habilita (on) o deshabilita (off) el autocompletado
  • autofocus - Atributo booleano que coloca el foco automáticamente en este campo al cargar la página
  • wrap - Controla cómo se envía el texto con saltos de línea (soft o hard)
  • spellcheck - Habilita (true) o deshabilita (false) la corrección ortográfica del navegador

Ejemplo práctico

<!-- Textarea básico -->
<label for="comentario">Comentario:</label>
<textarea id="comentario" name="comentario" rows="5" cols="50">
</textarea>

<!-- Textarea con placeholder y límite -->
<textarea name="mensaje" 
          placeholder="Escribe tu mensaje (máximo 500 caracteres)" 
          maxlength="500" 
          rows="8" 
          required>
</textarea>

<!-- Textarea con contenido predeterminado -->
<textarea name="descripcion" rows="6">
Este es el contenido inicial que aparece 
cuando se carga la página.
</textarea>

<!-- Textarea de solo lectura -->
<textarea name="terminos" rows="10" readonly>
Aquí van los términos y condiciones que el usuario
puede leer pero no modificar...
</textarea>

Notas importantes

  • Requiere etiqueta de cierre </textarea>
  • El contenido inicial va entre las etiquetas, no en un atributo value
  • Los usuarios pueden redimensionar el textarea (se puede controlar con CSS resize)
  • Siempre asociar con <label> para accesibilidad
  • Los espacios y saltos de línea entre las etiquetas se preservan
  • Es un elemento en línea por defecto, pero se comporta como bloque
  • Mejor controlar dimensiones con CSS que con rows/cols en diseño moderno
