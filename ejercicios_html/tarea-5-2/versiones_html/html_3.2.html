HTML 3.2 (1997)

Volver al inicio

HTML 3.2, publicado en enero de 1997 por el W3C, fue un estándar clave que formalizó características populares desarrolladas por navegadores como Netscape e Internet Explorer. Introdujo soporte para tablas, texto fluyendo alrededor de imágenes, subíndices/superíndices y Applets de Java, buscando mayor practicidad.

Aspectos clave de HTML 3.2 (1997)

Contexto:

Fue una solución temporal tras el borrador del HTML 3.0, buscando estandarizar el caos existente, y fue un paso previo a la versión 4.0.

Nuevas características:

Añadió tablas (<table>), <applet> para Java, control de fuentes, y mejor manejo de imágenes.

Formato de elementos:

Introdujo etiquetas para definir la presentación, como <font>, <center>, y <big>, muchas de las cuales fueron obsoletas posteriormente.

Compatibilidad:

Se diseñó para ser totalmente compatible con la versión anterior, HTML 2.0.

Autoría:

Fue desarrollado en colaboración con empresas líderes de la época, incluyendo Microsoft, Netscape y IBM.

A pesar de que el formato de elementos de presentación fue desaprobado en favor de las hojas de estilo (CSS) en versiones posteriores, HTML 3.2 definió la estética de la red a finales de los años 90.
